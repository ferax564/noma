What it doesA document pipeline built for people and agents.

Noma is small on purpose: human-readable source goes through a typed AST, then fans out to rich artifacts, validation, patch operations, and compact LLM context.

Readable source

Write Markdown-like prose plus explicit directive blocks such as ::claim, ::evidence, ::grid, ::plot, and ::agent_task.

Typed structure

The parser produces a stable AST with block IDs, aliases, positions, frontmatter, tables, lists, code, sections, and open-ended directives.

Beautiful outputs

Render standalone HTML, multi-page sites, PDFs, JSON AST, deterministic LLM text, and roundtrip-safe Noma source from the same file.

Validation

Catch duplicate IDs, broken wikilinks, stale citations, plot/data mismatches, unsafe escape hatches, missing evidence, incomplete risks, and profile violations.

Block-level patches

Agents can update one block, one body, one heading, or one attribute while preserving unrelated bytes and appending a transcript record.

Agent context

Export clean LLM context with --select, --exclude, --budget, and stale-aware memory filtering.

WorkflowThe safe loop for living documents.

The same commands work for one local memo, a book manifest, or an agent integration. Read structured context, discover IDs, patch the smallest target, validate, and render a publishable artifact. 

noma ids memo.noma
noma render memo.noma --to llm --select claim,evidence,risk
noma check memo.noma
noma patch memo.noma --ops patch.json --inplace
noma render memo.noma --to html --strict --out memo.html
01

Discover the stable surface

IDs and aliases are the API. Agents target blocks by ID instead of guessing line numbers.

02

Choose the smallest patch

Use replace_body, update_heading, or update_attribute before replacing whole blocks.

03

Validate before and after

Diagnostics make stale sources, broken references, unsafe blocks, and incomplete semantic content explicit.

04

Publish strict artifacts

Render HTML, PDF, book sites, or LLM context with the same source and a clear safety mode for review.

The three-layer modelOne source. Three audiences.

Markdown and HTML conflate what humans write, what humans read, and what machines parse. Noma separates them — and renders all three from a single source file.

01

Source

What humans and agents edit. Plain-text, clean Git diffs, stable over time, easy to patch at the block level.

02

Artifact

What humans consume. Standalone HTML by default — visual hierarchy, grids, cards, charts, copy buttons, mobile-responsive.

03

Agent context

What AI systems read. Deterministic LLM context — more structured than Markdown, less noisy than HTML.

Why NomaBetter source than HTML. Richer than Markdown.

Markdown wins on simplicity but lacks structure. HTML wins on rendering but collapses as source. Noma keeps the strengths of both, drops the weaknesses.

The central design testThree questions every feature must pass.

  1. Can a human understand this in raw text?
  2. Can an agent edit this safely without destroying the document?
  3. Can it render into an artifact people actually want to read?

If any answer is no, the feature does not belong in core Noma.
